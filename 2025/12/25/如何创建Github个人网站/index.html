坑 1：执行 hexo d 报错 “Deployer not found: git”

执行 hexo clean && hexo d 准备部署时，终端直接抛出 “Deployer not found: git” 错误，部署直接终止。

坑 2：部署时报错 “git@github.com: Permission denied (publickey)”

解决了部署器问题后，再次执行 hexo d，又遇到了权限拒绝错误。终端显示 “Permission denied (publickey)”，无法读取远程仓库。

原因分析

我最初配置的 repository 地址是 SSH 格式（git@github.com:...），但我的 Windows 电脑没有配置 SSH 密钥，也没有将密钥添加到 GitHub 账户中，导致 GitHub 拒绝了我的访问请求。

解决方案（两种方案可选）

方案 1：改用 HTTPS 地址（新手推荐，简单快捷）

将 _config.yml 中 deploy 的 repository 地址改为 HTTPS 格式（可在 GitHub 仓库页面复制）：

重新执行部署命令后，会提示输入 GitHub 用户名和 个人访问令牌（PAT）（GitHub 2021 年后不再支持密码登录，PAT 生成路径：GitHub 头像 → Settings → Developer settings → Personal access tokens，勾选 repo 权限后生成）。

方案 2：配置 SSH 密钥（一劳永逸）

  1. 检查本地是否有 SSH 密钥：ls -al ~/.ssh，若没有 id_rsa（私钥）和 id_rsa.pub（公钥），则执行下一步；

  2. 生成 SSH 密钥（替换为你的 GitHub 绑定邮箱）：

    按 3 次回车默认配置；

  3. 复制公钥内容：cat ~/.ssh/id_rsa.pub，复制输出的全部内容；

  4. 添加到 GitHub：GitHub 头像 → Settings → SSH and GPG keys → New SSH key，粘贴公钥并保存；

  5. 验证连接：ssh -T git@github.com，出现 “Hi 用户名! You’ve successfully authenticated…” 说明配置成功。

坑 3：部署成功后访问 GitHub Pages 显示 404

终于解决了权限问题，部署日志显示成功，但访问 myjathan.github.io 时却出现 404 错误。

原因分析

  1. 仓库名称不符合规范：个人主页仓库名必须是 用户名.github.io 格式，我初期设为 mayujie.github.io（与用户名 myjathan 不匹配）；

  2. 未配置 Pages 部署分支：虽然静态文件推到了 main 分支，但未在 GitHub 仓库设置中指定部署分支。

解决方案

  1. 修改仓库名：进入 GitHub 仓库 → Settings → Repository name，改为 myjathan.github.io（对应你的用户名），保存；

  2. 配置 Pages 部署分支：仓库 Settings → Pages（左侧 Code and automation 分类下），选择 Source 为 “Deploy from a branch”，Branch 选择 main，目录选根目录 /，点击 Save；

  3. 耐心等待：GitHub Pages 部署需要 1-5 分钟，等顶部出现绿色提示 “Your site is published at …” 就说明生效了。

坑 5：GitHub Actions 配置与手动部署冲突

添加 GitHub Actions 配置后，部署出现混乱。

原因分析

Actions 配置是“源码推 main → 云端构建 → 推 gh-pages 部署”，而我之前是“手动推静态文件到 main 部署”，两种方式逻辑冲突。

解决方案（二选一）

方案 1：继续手动部署（适合新手）

删除根目录下的 .github/workflows/deploy.yml 文件，保持手动流程：hexo clean && hexo d。

方案 2：切换到 Actions 自动部署（更规范）

  1. 将博客源码（而非静态文件）推到 main 分支；

  2. 在 GitHub 仓库 Settings → Pages 中，将部署分支改为 gh-pages；

  3. 保留 Actions 配置文件，后续推送源码到 main 分支时，Actions 会自动构建并部署。

三、总结：新手搭建博客的核心注意事项

经过这次踩坑之旅，我总结出几个新手必看的注意事项：

  1. 插件和依赖是基础：Hexo 的很多功能（如 Git 部署、主题渲染）都依赖额外插件，安装主题后一定要检查并安装对应的依赖；

  2. 配置文件要细心：YAML 格式对缩进和空格敏感，_config.yml 中的冒号后必须有空格，缩进用 2 个空格（不要用 Tab）；

  3. GitHub Pages 规范要记牢：个人主页仓库名必须是 用户名.github.io，部署分支要在 Settings 中明确指定，部署后需等待几分钟生效；

  4. 部署方式选一种即可：手动部署和 Actions 自动部署不要混用，避免分支冲突；

  5. 遇到问题先看日志：终端报错和 GitHub Actions 日志是排查问题的关键，很多错误信息能直接指向原因。

虽然过程踩了很多坑，但当最终在浏览器中看到自己的博客正常访问时，那种成就感还是很强的。这篇文章作为我的第一篇博客，既是对自己搭建过程的复盘，也希望能给新手朋友提供一些帮助。后续我还会分享更多 Hexo 美化、插件使用的教程，欢迎大家关注～
